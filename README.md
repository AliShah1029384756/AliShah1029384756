## How to Build Your Own Project - Step by Step

### Project Setup Checklist
```
1. Create project folder: mkdir my-project
2. Initialize Git: git init
3. Create folder structure:
   - /src - source files
   - /public - static files
   - /styles - CSS files
   - README.md
4. Start coding and commit regularly
```

### Best Practices to Follow
- **Write Clean Code**
  - Use meaningful variable names
  - Keep functions small and focused
  - Add comments for complex logic
  - Follow consistent indentation

- **Proper Error Handling**
  - Use try-catch for error-prone code
  - Validate user input
  - Display helpful error messages
  - Log errors for debugging

- **API Design Principles**
  - Use proper HTTP methods (GET, POST, PUT, DELETE)
  - Return meaningful HTTP status codes
  - Follow REST conventions
  - Document your endpoints

- **Database Best Practices**
  - Normalize your data structure
  - Use indexes for frequently queried fields
  - Validate data before storing
  - Backup your data regularly

- **Security Considerations**
  - Sanitize user input
  - Use environment variables for secrets
  - Hash passwords never store plain text
  - Implement proper authentication

## Common Mistakes to Avoid

❌ **Don't:**
- Leave code without comments or documentation
- Use generic variable names like `x`, `data`, `result`
- Hardcode sensitive information (API keys, passwords)
- Skip error handling "for now"
- Ignore responsive design
- Write extremely long functions (>50 lines)
- Push without committing meaningful messages

✅ **Do:**
- Write self-documenting code
- Test edge cases and invalid inputs
- Use `.env` files for configuration
- Handle errors proactively
- Mobile-first development approach
- Break code into reusable functions
- Write descriptive commit messages

## Troubleshooting Guide

### Common Issues & Solutions

**Issue: "Port already in use" error**
```
Solution: Kill the process using the port
Windows: netstat -ano | findstr :3000
        taskkill /PID <PID> /F
Mac/Linux: lsof -ti:3000 | xargs kill -9
```

**Issue: "Module not found" error**
```
Solution: Install missing dependencies
npm install
or
npm install <package-name>
```

**Issue: Git changes not showing**
```
Solution: Check git status
git status
git add .
git commit -m "Your message"
```

**Issue: Fetched data not displaying**
```
Check browser console (F12) for errors
Verify API endpoint URL
Check network tab for API response
```

**Issue: CSS not applying**
```
Clear browser cache (Ctrl+Shift+Delete)
Check CSS file path
Verify CSS selector specificity
Use browser DevTools to inspect elements
```

### Browser DevTools Like a Pro
- **F12** - Open DevTools
- **Console Tab** - Debug JavaScript errors and logs
- **Elements Tab** - Inspect and modify HTML/CSS live
- **Network Tab** - Monitor API requests

### Debugging Strategies
```javascript
// Use console methods effectively
console.log('Value:', variable);
console.table(arrayData);
console.error('Error message:', error);
console.time('label'); // measure performance
console.timeEnd('label');

// Add breakpoints in debugger
debugger; // execution pauses here
```

### Naming Conventions
- **Variables**: camelCase (`userName`, `isActive`)
- **Constants**: UPPER_SNAKE_CASE (`API_KEY`, `MAX_RETRIES`)
- **Classes**: PascalCase (`UserProfile`, `DatabaseConnection`)
- **Functions**: camelCase verbs (`fetchData()`, `handleSubmit()`)
